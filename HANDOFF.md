# Relay ML Handoff

Current status: **fully functional ML service** with Bedrock-backed image grading (default),
multi-angle image grading, CNN fallback, keyframe-based video grading, Bayesian MultiFlags,
embedding endpoint (local or Bedrock Titan), wish-score, and return-reason clustering.

**Tests: 55 passing**

---

## Endpoints (complete list)

| Method | Path | Description |
|---|---|---|
| GET | `/health` | Liveness + model/mode status |
| POST | `/grade-image` | Grade single image → ConditionPassport |
| POST | `/grade-images` | Grade 1-8 images of same product (multi-angle) → ConditionPassport |
| POST | `/grade-video` | Grade video via keyframe extraction (Bedrock multi-image in bedrock_only mode) |
| POST | `/fit-flags` | Aggregate-backed fit flags for a category/SKU |
| POST | `/fit-flags/multi` | Bayesian MultiFlags from raw return counts |
| POST | `/embed` | 384-d embedding vector (local or Bedrock Titan) |
| POST | `/wish-score` | Buyer-intent confidence score (0-1) |
| POST | `/return-clusters` | NLP clustering of free-text return reasons |

---

## Grading Modes (`GRADING_MODE` env var)

| Mode | What it does | When to use |
|---|---|---|
| `bedrock_only` **(default)** | Grades via AWS Bedrock Nova Lite — no CNN needed | Production / demo (most accurate) |
| `cnn` | Local CNN first, escalates to Bedrock T2/T3 on low confidence | When you want fast local inference |
| `mock` | Deterministic stub passport (grade B, no AI) | relay-api local dev before ML URL is live |

**Key**: Bedrock mode correctly handles undamaged products (returns grade A+/A with zero defects).
CNN mode is biased toward predicting damage due to defect-heavy training data.

## What Shikher Needs To Know

### Integration URL
```
ML_SERVICE_URL=http://localhost:8001
```

### Embedding dimension = 384
- Matches `EMB_DIM = 384` in relay-api's Alembic migration
- Works with both local (sentence-transformers) and Bedrock Titan backends
- No pgvector schema change needed

### New endpoint: `POST /grade-images` (multi-angle)
```
images: list of 1-8 JPEG/PNG files (multipart)
unit_id: string
category: string
return_id: optional string
```
Returns a single ConditionPassport assessed from all angles together.
`model_tier_used` = `"bedrock-only+Nangles"` (e.g. `bedrock-only+4angles`).

### Wish-score contract
```json
POST /wish-score
{"wish_age_days": 3, "user_purchase_count": 5, "category_affinity": 0.8, "has_fit_profile": true}
→ {"score": 0.94, "model": "logreg_v1"}
```

### Embed contract
```json
POST /embed
{"text": "red leather jacket size M"}
→ {"vector": [384 floats], "model": "bedrock-titan-v2"}
```
Also accepts structured: `{"category": "dress", "grade": "B", "size": "M", "vertical": "fashion"}`

### MultiFlags contract
```json
POST /fit-flags/multi
{"sku_id": "SKU-1", "category": "dress", "total_returns": 100, "too_small_count": 40, "too_large_count": 5, "fit_count": 55, "is_article_level": true}
→ {"sku_id": "SKU-1", "flags": [{"type": "runs_small", "message": "...", "confidence": 0.82}], "source": "multiflags_v1:article"}
```

### Return-clusters contract
```json
POST /return-clusters
{"reasons": ["color not as shown", "shade is different", "too small", "size runs small"], "n_clusters": 2}
→ {"clusters": [...], "total_reasons": 4, "num_clusters": 2}
```

## Docker Build Note

If using `EMBEDDING_MODEL=bedrock-titan` and `GRADING_MODE=bedrock_only`:
- `torch`, `timm`, and `sentence-transformers` are NOT needed at runtime
- Only `boto3`, `fastapi`, `pydantic`, `pillow`, `scikit-learn` are required
- Dramatically faster Docker builds (~30s vs ~5min with torch)

For minimal Dockerfile, create `requirements-lite.txt` without torch/timm/sentence-transformers.

## Current Caveats

- `models/grade_cnn_v1.pt` is git-ignored. Only needed if `GRADING_MODE=cnn`.
- CNN v1 only predicts 3 defect classes (`crack`, `damaged`, `other`). Bedrock gives richer results.
- Bedrock mode requires network access to AWS (not offline-capable).
- wish-score model trained on synthetic data — retrain on real conversion data when available.
